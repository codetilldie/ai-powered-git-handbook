## 第一章：Git 入门

欢迎来到 Git 的世界！本章将引导你完成从零到一的整个过程：安装、配置，并完成你的第一次代码提交。读完本章，你将对 Git 的基本工作流程建立起坚实的理解。

### 1.1 什么是 Git？

Git 是一个**分布式版本控制系统 (Distributed Version Control System, DVCS)**。简单来说，它是一个能帮你做这两件事的工具：

1.  **追踪文件变化**：Git 能记录你项目里每一个文件的每一次改动。你可以随时回头查看任何一个历史版本，就像拥有了一台时光机。
2.  **团队协作**：Git 能让多个人在同一个项目上高效工作，而不会互相干扰。它能帮你轻松合并大家的代码，并清楚地记录下谁、在什么时候、做了什么修改。

与传统的集中式版本控制系统（如 SVN）不同，Git 的“分布式”特性意味着每个开发者的电脑上都有一个完整的代码仓库（包含所有历史记录）。这使得大部分操作（如提交、查看历史）都可以在本地快速完成，无需联网。

### 1.2 Git 的安装与配置

#### 安装

- **Windows**: 访问 [git-scm.com](https://git-scm.com/download/win) 下载安装包，一路“下一步”即可。安装时建议勾选 "Git Bash Here"，它会提供一个强大的类 Linux 终端。
- **macOS**: 打开“终端 (Terminal)” 应用，输入 `git --version`。如果未安装，系统会自动提示你安装“命令行开发者工具”，按提示操作即可。或者，你也可以通过 [Homebrew](https://brew.sh/) (`brew install git`) 来安装。
- **Linux**: 打开终端，根据你的发行版输入相应命令：
  - Debian/Ubuntu: `sudo apt-get install git`
  - Fedora: `sudo dnf install git`

#### 首次配置

安装完成后，你需要告诉 Git 你是谁。这个信息会出现在你的每一次提交记录里。

打开终端（Windows 用户建议使用 Git Bash），输入以下命令：

```bash
# 设置你的用户名
git config --global user.name "Your Name"

# 设置你的邮箱地址
git config --global user.email "youremail@example.com"

# 设置默认分支名为 master
git config --global init.defaultBranch master
```

`--global` 参数表示这是全局配置，这台电脑上的所有 Git 仓库都会使用这个配置。

### 1.3 开启你的 Git 之旅：`init` 与 `clone`

有两种方式可以开始一个 Git 项目：

#### 方式一：`git init` (初始化新仓库)

用于在一个**已存在的项目**或**空目录**中创建全新的 Git 仓库。

```bash
# 进入你的项目目录
cd my-project

# 初始化仓库
git init
```
Git 会在当前目录下创建一个 `.git` 隐藏目录，我们称之为“Git 仓库”。所有版本历史、配置等都存储在这里。

*(关于 `.git` 目录的详细剖析，请参见第七章)*

#### 方式二：`git clone` (克隆现有仓库)

如果你想参与一个已经存在的项目（例如 GitHub 上的开源项目），你需要将它克隆到本地。

```bash
# 克隆一个远程仓库到你的电脑
git clone https://github.com/git/git.git

# 这会在当前目录下创建一个名为 "git" 的新文件夹
# 其中包含了项目的所有文件和完整的版本历史
```

### 1.4 核心工作流：三大区域

这是 Git 最核心、最需要理解的概念。Git 管理的项目有三个主要区域：

1.  **工作区 (Working Directory)**: 你电脑上能直接看到和编辑的项目文件夹。
2.  **暂存区 (Staging Area / Index)**: 一个特殊的文件，位于 `.git` 目录中。它像一个“购物篮”，记录了你**希望**在下一次提交中包含哪些文件的哪些变更。
3.  **仓库 (Repository)**: 位于 `.git` 目录中，是永久存储你项目所有版本快照的地方。

工作流程就像这样：
`工作区` -> (执行 `git add`) -> `暂存区` -> (执行 `git commit`) -> `仓库`

### 1.5 基础命令：`status`, `add`, `commit`, `log`, `diff`

这五个命令构成了 Git 最日常的工作循环。

1.  **`git status`**: **查看状态**
    - 这是最常用的命令！它告诉你当前工作区和暂存区有哪些文件被修改、新增或删除了。
    - **红色**表示文件在工作区有改动，但还未放入暂存区。
    - **绿色**表示文件已放入暂存区，等待被提交。

2.  **`git add <文件名>`**: **添加到暂存区**
    - 这个命令将工作区的指定文件变更“装进”暂存区的“购物篮”。
    - `git add .` 是一个常用命令，表示将当前目录下所有变更都添加到暂存区。

3.  **`git commit -m "提交信息"`**: **提交到仓库**
    - 这个命令会取出暂存区的所有内容，创建一个新的“版本快照”（即一次提交），并将其**永久**保存在仓库中。
    - `-m` 参数后面跟着的是本次提交的说明信息，**必须填写**，且应清晰明了。例如 `"feat: 添加用户登录功能"`。

4.  **`git log`**: **查看提交历史**
    - 它会按时间倒序列出所有的提交记录，包括提交的 SHA-1 哈希值、作者、日期和提交信息。

5.  **`git diff`**: **查看差异**
    - `git diff`: 查看**工作区**相对于**暂存区**的差异。
    - `git diff --staged`: 查看**暂存区**相对于**上一次提交**的差异。
    - `git diff HEAD`: 查看**工作区**相对于**上一次提交**的差异。

通过这套“查看状态 -> 添加到暂存区 -> 提交到仓库”的流程，你就完成了 Git 最基本的工作循环。
